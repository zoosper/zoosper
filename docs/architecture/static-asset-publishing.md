# Phase 0.61 - Static asset publishing foundation

## Goal

Make frontend theme CSS/JS/assets available under `public/static` so CDN/static URLs generated by `$cdn->staticAsset()` resolve in the browser.

## Why this was needed

Phase 0.58 correctly generated URLs such as:

```text
/static/themes/default/assets/css/app.css
```

The homepage loaded without style because the source CSS existed under `themes/default/assets/css/app.css`, but the published public asset did not exist under `public/static/themes/default/assets/css/app.css`.

## Initial publishing model

```text
themes/<theme>/assets/*
  -> public/static/themes/<theme>/assets/*
```

A future phase can replace this simple copier with a Vite/Tailwind build pipeline.
