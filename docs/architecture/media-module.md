# Media Module Foundation

Phase 1.37 introduces `Zoosper_Media`, the first media/file-management foundation module.

## Security model

- Originals are stored outside the public webroot under `storage/media/original`.
- Browser-facing files are copied under `public/media` only after upload validation succeeds.
- The first version allows only JPG, PNG, GIF and WebP images.
- SVG, PHP, archives, documents and other executable or complex formats are rejected until dedicated sanitisation policies exist.
- Upload paths are generated by the server; user filenames are never used as filesystem paths.
- Path traversal is rejected.

## Current scope

Phase 1.37 provides the media library table, admin upload/list screens, validation, private storage and controlled public publishing.

It does not yet wire Editor.js image blocks. That should happen after the media asset foundation verifies green.
