# Zoosper technical documentation website roadmap

## Purpose

Start writing developer/user docs as features are built so the future Zoosper documentation website can be generated from real, current architecture notes.

## Suggested sections

```text
Getting started
Installation
Configuration
Admin guide
Developer guide
Module development
Composer marketplace modules
Themes and templates
Site/store-view context
CDN and cache strategy
Security and PCI-aware logging
2FA and SMTP setup
Troubleshooting
API reference
```

## Current docs to reuse

The `docs/architecture`, `docs/operations`, `docs/progress`, `docs/developer` and `docs/roadmap` folders should become the content source for the future docs site.

## Rule going forward

Every implementation phase should add or update technical documentation alongside code so knowledge is not lost.
